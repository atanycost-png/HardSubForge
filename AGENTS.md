# AGENTS.md

## Overview

HardSubForge is a PySide6 desktop GUI app that re-encodes video with hardsubbed subtitles, watermarks, and audio track selection. It wraps FFmpeg for all video processing. No test suite, no linter, no type checker, no CI.

## Entrypoint & architecture

- `python main.py` — PySide6 `QApplication`, creates `MainWindow`
- Six packages — **no shared code between them**, each is self-contained:
  - `config/` — `config.json` persistence (`ConfigManager`)
  - `ffmpeg/` — FFmpeg command builder + subprocess runner (`FFmpegWrapper`)
  - `presets/` — built-in `QualityPreset` dataclasses + `CustomPreset` support
  - `ui/` — `MainWindow` + reusable widgets (`DropArea`, `AddPresetDialog`, `SectionCard`, `StatusPill`), style tokens (`styles.py`), theme manager (`theme.py`)
  - `workers/` — `QObject`-based workers (`ConversionWorker`, `ProbeWorker`) moved to `QThread` via `moveToThread()`
  - `utils/` — stateless helpers (binary detection, text escaping, GPU check)

## Setup

```bash
pip install -r requirements.txt   # PySide6>=6.6.0, pyinstaller>=6.0.0
python main.py
```

**External dependency**: FFmpeg and FFprobe must be on PATH **or** `ffmpeg.exe`/`ffprobe.exe` placed in the project root directory. The app auto-detects in this order: bundled in root > `shutil.which("ffmpeg")` > shows warning dialog. The "download automatically" button is a **stub** (`_download_ffmpeg_windows` is `pass`).

## Build

```bat
build.bat   # PyInstaller via HardSubForge.spec
```

The `.spec` file is **gitignored** (`*.spec`) but `build.bat` expects `HardSubForge.spec` to exist. It is typically generated by a prior PyInstaller run or via `pyi-makespec`. The build cleans `build/` and `dist/` first, then creates `dist/HardSubForge.exe`.

## Critical quirks

### Threading model (v3.0.1+)
- Workers use the **QObject + moveToThread** pattern (not QThread subclass):
  - `ConversionWorker(QObject)` — moved to `QThread`, communicates via signals only
  - `ProbeWorker(QObject)` — moved to `QThread`, same pattern
  - Cancellation uses `QMutex` + `QMutexLocker` for thread safety
  - Thread cleanup: `worker.finished` → `thread.quit` → `worker.deleteLater` + `thread.deleteLater`

### FFmpeg subprocess behavior
- On Windows, all `subprocess` calls use `CREATE_NO_WINDOW` (`creationflags=subprocess.CREATE_NO_WINDOW`) to suppress console popups. **Do not remove this flag.**
- FFmpeg stdout **and** stderr are merged into stdout (`stderr=subprocess.STDOUT`), so the conversion log parser reads a single stream.
- Progress parsing uses pre-compiled regex at module level (`TIME_PATTERN`, `DURATION_PATTERN`). Use `"substring" in line` before calling `.search()` — this is intentional for CPU savings in the hot loop.
- Progress signals are **throttled**: only emitted when the integer percent changes. Log signals filter out `"time="` lines to avoid flooding the UI thread. The log widget is capped at 1000 blocks (`setMaximumBlockCount`).

### Style system (v3.0.1+)
- `ui/styles.py` — Color palette (`Color`), spacing (`Spacing`), border radius (`Radius`) as frozen dataclasses
- `ui/theme.py` — `apply_dark_theme(app)` applies Fusion + global QSS stylesheet
- All widgets use style constants; never hardcode color hex values inline
- `SectionCard` wraps card-like groups; `StatusPill` wraps pill-shaped status indicators

### GPU detection
- NVIDIA GPU detection calls `nvidia-smi` via subprocess. `check_nvidia_gpu()` is called at startup (in `MainWindow.__init__`) and on each conversion in `FFmpegWrapper._has_nvidia_gpu()`. These are **independent** calls — don't assume they stay in sync.
- If NVENC is unavailable or hardware accel is disabled, falls back to `libx264` with `preset=medium` (hardcoded).

### Custom presets
- Stored inside `config.json` as a JSON array under `"custom_presets"`. Each entry is a dict serialized from `CustomPreset.to_dict()`.
- Distinguished in the combo box by `⚙️` prefix (built-in presets use `🎬`). The delete logic checks the prefix to block deletion of built-in presets.

### Font detection
- Platform-specific hardcoded font paths (Arial on Windows, DejaVuSans on Linux, etc.). Uses `lru_cache(maxsize=1)`. If no font is found, watermark/subtitle rendering silently fails — no error is raised.

### Binary path caching
- `get_ffmpeg_binary()` and `get_font_path()` use `lru_cache(maxsize=1)`. This means a manual FFmpeg path change via the settings dialog **recreates the FFmpegWrapper** (see `_select_ffmpeg_manual` in `main_window.py:697`), but the cached helper is not invalidated. If you add runtime binary switching, you must recreate the wrapper instance.

### Subtitle detection is filename-only
- `_detect_subtitle()` only checks for `.srt`/`.ass`/`.ssa` files with the **same stem** as the video in the **same directory**. It does not probe embedded subtitles (that feature is planned for v3.1).

### Audio probing
- Runs `ffprobe` in a `ProbeWorker` QObject on a `QThread`. Returns a list of dicts with `index`, `title`, `codec`, `language`. Language is mapped via `LANGUAGE_NAMES` dict. Unknown codes fall through as-is.

### PyInstaller frozen path
- `get_ffmpeg_binary()` checks `sys._MEIPASS` for bundled binaries first. `_MEIPASS` is only set when running as a PyInstaller executable — in dev mode, it falls back to `Path(__file__).parent.parent`.

### No embedded subtitles or batch processing
- Embedded subtitle extraction and batch conversion were **removed** in v3.0.0 (see CHANGELOG). They are planned for v3.1. Do not add them back using old code paths.

## Style notes

- Codebase is in Portuguese (docstrings, user-facing strings, comments). Variable names are in English.
- Follow PEP 8. No pre-existing formatter config.
- `config.json` is gitignored — never commit user settings.
- Use `@Slot()` decorator on all slot methods. Signals should use Qt 6 scoped enums (`Qt.AlignmentFlag.AlignCenter`, not `Qt.AlignCenter`).
